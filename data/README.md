📊 Estrutura de Dados
Os dados deste projeto são sintéticos e gerados internamente pelo notebook para simular um ambiente de transações financeiras.

**- Total de colunas:** 31 (Variáveis V1-V28, Amount, Class e Time).
**- Desbalanceamento:** 99,1% de transações normais vs 0,9% de fraudes.
**- Objetivo técnico:** Tratar o baixíssimo Recall inicial (0.00 nas métricas de baseline) para conseguir identificar a classe 1 (fraude).
